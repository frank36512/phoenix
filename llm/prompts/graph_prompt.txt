你是一位**资深知识图谱建模专家**。请基于提供的主题和动画内容摘要，构建一个**逻辑严密、关系清晰**的知识图谱。

**输入信息：**
*   **核心主题**：{topic}
*   **内容参考**：{animation_slice}

**构建目标：**
提取主题下的核心概念实体及其相互关系，形成一个网络化的知识结构，帮助用户理解概念间的关联。

**生成要求：**
1.  **节点 (Nodes)**：
    *   数量：8-15 个。
    *   字段：
        *   `id`: 唯一标识符（字符串）。
        *   `label`: 显示名称（中文，简练，不超过 6 个字，**不要包含标点符号**）。
        *   `group`: 分组编号（1=核心主题, 2=关键概念, 3=组成要素, 4=应用/实例, 5=价值/意义）。
2.  **边 (Edges)**：
    *   数量：10-20 条（确保图谱连通性好，避免孤立节点）。
    *   字段：
        *   `from`: 起始节点ID。
        *   `to`: 目标节点ID。
        *   `label`: 关系描述（动词或介词短语，如“包含”、“导致”、“应用于”、“属于”，**不要包含标点符号**）。
3.  **质量控制**：
    *   关系必须真实准确，符合科学或逻辑事实。
    *   避免循环依赖（A->B->A）除非逻辑上确实如此。
    *   核心主题节点必须处于网络的中心位置（连接度最高）。
    *   包含“价值/意义”类节点，体现主题的升华（对应动画的结尾总结）。

**输出格式：**
严格输出标准 JSON 格式，**不要包含** Markdown 代码块标记（```json ... ```），只输出纯 JSON 字符串。

**JSON 结构示例：**
{{
  "nodes": [
    {{ "id": "n1", "label": "光合作用", "group": 1 }},
    {{ "id": "n2", "label": "叶绿体", "group": 3 }},
    {{ "id": "n3", "label": "氧气", "group": 3 }}
  ],
  "edges": [
    {{ "from": "n1", "to": "n2", "label": "发生场所" }},
    {{ "from": "n1", "to": "n3", "label": "产物" }}
  ]
}}
